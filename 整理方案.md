# 📁 Code Space 目录整理方案

> 生成时间: 2026-06-28

## 🔍 当前问题分析

### ❌ 发现的问题

1. **目录结构混乱**
   - 根目录混合了多种类型项目
   - 命名不规范（"一些工具"、"清单"等）

2. **体积过大**
   - `wo-laibang-app/node_modules/` 未被git忽略
   - `PC-action-macOS/recordings/` 有18个录制文件夹

3. **文件散落**
   - 项目目录混入非项目文件（`chat模式下的.txt`）
   - 工具文件未统一管理

## 📋 整理步骤清单

### 步骤1: 创建新目录结构

在 `code空间` 根目录下创建以下文件夹：
- `01-开发项目`
- `02-Web应用`
- `03-移动应用`
- `04-工具脚本`
- `05-文档资源`
- `06-临时文件`

### 步骤3: 删除空目录

删除已经清空的 `项目/` 目录

### 步骤4: 更新 .gitignore

新的 `.gitignore` 已包含以下规则：

```gitignore

# Node (重要！)

# 临时文件

*.log
*.bak

# OS
.DS_Store
Thumbs.db

# Facetune
*.fac
```

## 🔧 后续维护建议

### 每周清理
1. 检查 `06-临时文件/` 是否有需要归档的文件
2. 清理不需要的录制文件
3. 更新README.md

### 新项目规范
- 开发项目 → `01-开发项目/`
- Web页面 → `02-Web应用/`
- App项目 → `03-移动应用/`
- 工具脚本 → `04-工具脚本/`

### Git提交前检查
```bash
# 检查是否有大文件
git ls-files | xargs du -sh | sort -rh | head -10

# 检查是否忽略了node_modules
git status wo-laibang-app/node_modules
```
